# 第一部分：MindShow 快速上手指南

## 1.1 准备工作

1. 打开浏览器，访问：https://www.mindshow.fun
2. 用手机号/微信注册并登录（免费版够用）
3. 准备素材：
   - 本文件中的 PPT 内容（已按页整理好）
   - 项目截图（见第五部分「配图素材清单」）
   - 学校/课程要求的 PPT 模板（如有）

## 1.2 导入 Markdown 生成 PPT

1. 点击首页 **「创建PPT」**>**「导入Markdown」**
2. 复制本文档 **「第二部分：逐页 PPT 内容」** 中的 Markdown 文本（从第 1 页到第 24 页）
3. 粘贴到 MindShow 的 Markdown 编辑框
4. 点击 **「生成PPT」**
5. MindShow 会自动按 `#` / `##` 分页，生成 24 页幻灯片

## 1.3 调整版式与模板

1. 在 MindShow 左侧选择 **「模板」**，挑一个简洁大方的学术/科技风模板
2. 建议：
   - 封面用深色或校园风背景
   - 内容页用浅色背景，文字清晰
   - 图表页选择有左右分栏或大图展示的版式
3. 逐页检查：
   - 标题是否完整
   - 表格是否被截断
   - 要点是否过多（一页不超过 6 行）

## 1.4 插入图片

1. 点击某页 PPT 中的图片占位区
2. 上传本地截图（建议 PNG/JPG，单张不超过 2MB）
3. 图片比例失调时，点击 **「裁剪」** 或 **「自适应」**
4. 关键截图建议放大展示，文字说明放右侧

## 1.5 导出 PPTX

1. 点击右上角 **「导出」**
2. 选择 **PPTX（可编辑）**
3. 下载后用 PowerPoint / WPS 打开
4. 最终检查字体、动画、分页、学校 Logo

## 第 20 页：部署踩坑与解决方案

### 页面内容

```markdown
# 部署踩坑与解决方案

## 5 个真实问题与排坑记录

| 问题 | 根因 | 解决方案 |
|------|------|----------|
| 镜像拉取超时 | Docker Hub 国内受限 | 配置 6 个国内镜像加速器 |
| Nacos 端口绑定失败 | 8848 在 Windows 保留端口段 | 宿主端口改为 8758 |
| 服务启动崩溃 | spring.config.import 无 dataId | 关闭配置中心，保留服务发现 |
| Redis 连接失败 | Boot 3.2 配置项为 spring.data.redis | 修正 redis 配置路径 |
| 网关路由 503 | 缺少 spring-cloud-loadbalancer | 网关 pom 增加依赖 |
```

### MindShow 操作建议

# 第四部分：MindShow 常见问题

| 问题 | 解决方案 |
|------|----------|
| 导入后分页不对 | 检查 Markdown 中 `#` / `##` 层级，确保每页用 `##` 开头 |
| 表格显示不全 | 选择「表格页」版式，或把表格拆成两页 |
| 图片上传失败 | 压缩图片到 2MB 以下，换 PNG/JPG 格式 |
| 免费版有水印 | 导出后用 PowerPoint 删除水印，或购买会员 |
| 模板不喜欢 | 先随便选一个，导出后去 PowerPoint 里换母版 |
