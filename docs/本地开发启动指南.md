# 本地开发启动指南（无 Docker）

> 适用于答辩前本地快速启动、截图、演示。
> 本模式使用 `dev`  profile，**不依赖 Nacos**，Gateway 直接路由到本地服务端口。

---

## 一、前置条件

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | Spring Boot 3.2 需要 |
| Maven | 3.8+ | 项目构建 |
| MySQL | 8.0+ | 端口 3306 |
| Redis | 6.0+ | 端口 6379 |
| Node.js | 18+ | 前端运行 |
| IDEA / VS Code | — | 启动后端服务 |

---

## 二、初始化数据库

1. 启动本地 MySQL 和 Redis
2. 创建数据库：
   ```sql
   CREATE DATABASE campus_studyroom CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 执行初始化脚本：
   ```bash
   mysql -uroot -p123456 campus_studyroom < test/sql/mysql/campus_studyroom.sql
   ```
   > 脚本已同步更新，包含 `study_room.description` 和 `study_seat` 表。

## 三、后端服务启动顺序

在 IDEA 中按以下顺序启动（每个服务对应一个端口）：

| 顺序 | 服务 | 端口 | 启动类 |
|------|------|------|--------|
| 1 | campus-auth | 8001 | `CampusAuthApplication` |
| 2 | campus-user | 8002 | `CampusUserServiceApplication` |
| 3 | campus-room | 8003 | `CampusRoomServiceApplication` |
| 4 | campus-reservation | 8004 | `CampusReservationServiceApplication` |
| 5 | campus-attendance | 8005 | `CampusAttendanceServiceApplication` |
| 6 | campus-ai | 8006 | `CampusAiServiceApplication` |
| 7 | campus-gateway | 8000 | `CampusGatewayApplication` |

### IDEA 启动参数

每个服务的 **Active profiles** 设置为：
```
dev
```

设置方式：
1. 打开 Run/Debug Configurations
2. 找到对应服务的启动配置
3. 在 **Environment → Active profiles** 填入 `dev`
4. 应用并启动

> 如果 IDEA 没有显示 profile 选项，可以在 **VM options** 里加：
> ```
> -Dspring.profiles.active=dev
> ```

## 四、启动前端

```bash
cd frontend
npm install
npm run dev
```

默认运行在：**http://localhost:5173**

## 五、验证

### 1. 网关健康

```bash
curl http://localhost:8000/actuator/health
```

预期：
```json
{"status":"UP"}
```

### 2. 登录

```bash
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student1","password":"123456"}'
```

预期：返回 `code:200` 和 `accessToken`。

### 3. 浏览器访问

打开 http://localhost:5173，用 `student1` / `123456` 登录。

## 六、与 Docker 部署的区别

| 项目 | Docker 模式 | 本地 dev 模式 |
|------|-------------|---------------|
| Nacos | 需要 | 不需要 |
| MySQL | Docker 容器 3307 | 本地 3306 |
| Redis | Docker 容器 6380 | 本地 6379 |
| 微服务端口 | 8000–8006 | 8000–8006 |
| 前端地址 | http://localhost | http://localhost:5173 |
| Gateway 路由 | Nacos 服务发现 | 直连 localhost |

## 七、常见问题

### 1. 端口被占用

如果 3306/6379/8000–8006 被占用，先关闭占用端口的程序，或修改 `application-dev.yml` 中的端口。

### 2. 数据库连接失败

检查 MySQL 是否启动、数据库是否存在、root 密码是否为 `123456`。

### 3. 某个服务起不来

按启动顺序检查前置服务是否已启动。Gateway 最后启动，依赖其他服务。

### 4. 前端调用接口 404

确认 Gateway 已启动，并且前端 `src/utils/request.ts` 中的 `baseURL` 为 `/api`。

## 八、快速命令

```bash
# 启动前端
cd frontend
npm run dev

# 测试登录
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student1","password":"123456"}'

curl http://localhost:8000/api/room/rooms?page=1\&size=12 \
  -H "Authorization: Bearer <token>"
```
