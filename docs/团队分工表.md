## 五、沟通与协作机制

1. **代码管理**：使用Gitee仓库，分支策略：main（稳定版）、dev（开发版）、feature/*（功能分支）
2. **文档管理**：docs目录下统一管理，Markdown格式
3. **接口协作**：后端先行定义接口文档（Knife4j），前端按文档对接
4. **每日同步**：每日沟通开发进度与问题
5. **代码审查**：合并前互相Review，确保代码质量

## 六、风险与应对

| 风险 | 应对措施 |
|------|----------|
| 达梦8数据库环境搭建困难 | 优先保证MySQL版完整，达梦8做兼容适配 |
| AI模型调用不稳定 | 设计降级策略，缓存历史推荐结果 |
| 前后端联调时间不足 | 尽早定义接口文档，并行开发 |
| K8s部署复杂度高 | 使用Docker Desktop内置K8s，简化部署 |
