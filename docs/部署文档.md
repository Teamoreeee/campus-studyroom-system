# 校园自习室预约系统 - 部署文档

## 一、本地开发环境

### 1. 前置依赖

- JDK 17+
- Maven 3.9+
- Node.js 20+
- MySQL 8.0+
- Redis 7.0+
- Nacos 2.3+

### 2. 初始化数据库

```bash
mysql -uroot -p123456 < test/sql/mysql/campus_studyroom.sql
```

### 3. 启动后端

```bash
mvn clean install -DskipTests
cd backend/campus-auth && mvn spring-boot:run
# 依次启动其他服务
```

### 4. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端默认访问：http://localhost:3000

## 二、Docker Compose 部署

### 1. 构建并启动全部服务

```bash
docker-compose up -d --build
```

### 2. 访问地址

- 前端：http://localhost
- 网关：http://localhost:8000
- Nacos：http://localhost:8848/nacos

### 3. 停止服务

```bash
docker-compose down
```

## 三、Kubernetes 部署

### 1. 构建镜像

```bash
# 构建所有镜像（需在 Docker Desktop K8s 或本地 Docker 环境）
docker build -t campus-studyroom/campus-auth:latest -f backend/campus-auth/Dockerfile .
docker build -t campus-studyroom/campus-user:latest -f backend/campus-user/Dockerfile .
docker build -t campus-studyroom/campus-room:latest -f backend/campus-room/Dockerfile .
docker build -t campus-studyroom/campus-reservation:latest -f backend/campus-reservation/Dockerfile .
docker build -t campus-studyroom/campus-attendance:latest -f backend/campus-attendance/Dockerfile .
docker build -t campus-studyroom/campus-ai:latest -f backend/campus-ai/Dockerfile .
docker build -t campus-studyroom/campus-gateway:latest -f backend/campus-gateway/Dockerfile .
docker build -t campus-studyroom/campus-frontend:latest -f frontend/Dockerfile .
```

### 2. 部署到 K8s

```bash
kubectl apply -f k8s/
```

### 3. 查看服务

```bash
kubectl get pods -n campus-studyroom
kubectl get svc -n campus-studyroom
```

### 4. 访问入口

- 前端：通过 `campus-frontend` 的 NodePort
- 网关：通过 `campus-gateway` 的 NodePort
- Prometheus：通过 `prometheus` 的 NodePort
- Grafana：通过 `grafana` 的 NodePort
- SkyWalking UI：通过 `skywalking-ui` 的 NodePort

## 四、监控说明

| 组件 | 地址 | 默认账号 |
|------|------|---------|
| Prometheus | http://localhost:9090 | - |
| Grafana | http://localhost:3000 | admin / admin123 |
| SkyWalking UI | http://localhost:8080 | - |

> 注意：后端服务需要添加 `spring-boot-starter-actuator` 依赖并开启 Prometheus 端点，才能被 Prometheus 正常采集。

## 五、常见问题

1. **Nacos 启动失败**：确保数据库中已创建 `nacos` 数据库并导入 Nacos 初始化 SQL。
2. **服务注册不上**：检查 `spring.cloud.nacos.discovery.server-addr` 配置是否正确。
3. **网关 404**：确认后端 Controller 路径与网关路由配置一致。
