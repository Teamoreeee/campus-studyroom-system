  - 前端 http://localhost → 200
  - Nacos http://localhost:8758/nacos/ → 200

  所以服务是正常的，问题可能在你的浏览器这边。

  ---
  试试这几个办法

  1. 用 127.0.0.1 代替 localhost

  有时候浏览器对 localhost 解析有问题：

  - 前端：http://127.0.0.1
  - Nacos：http://127.0.0.1:8758/nacos
  - K8s 前端：http://127.0.0.1:30081

  2. 换浏览器 / 隐身模式

  可能是缓存或插件问题。换个浏览器，或者按 Ctrl + Shift + N 开隐身窗口再试。

  3. 检查 URL 是否写错

  ┌──────────────┬─────────────────────────────┐
  │     目标     │          正确地址           │
  ├──────────────┼─────────────────────────────┤
  │ Docker 前端  │ http://localhost            │
  ├──────────────┼─────────────────────────────┤
  │ Docker Nacos │ http://localhost:8758/nacos │
  ├──────────────┼─────────────────────────────┤
  │ K8s 前端     │ http://localhost:30081      │
  ├──────────────┼─────────────────────────────┤
  │ K8s Nacos    │ http://localhost:9876/nacos │
  └──────────────┴─────────────────────────────┘

  4. 清 DNS 缓存

  以管理员身份打开 CMD，执行：
  ipconfig /flushdns

  然后再刷新浏览器。
